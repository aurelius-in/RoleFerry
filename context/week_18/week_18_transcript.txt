
Roleferry 3/3/26
Summary
Context
Product-development walkthrough comparing our Role Fairy app to SimplyApply.ai’s job-collection/recommendation product.
Main objective: decide whether to build job-collection / recommendation / auto-apply functionality in-house or integrate a third-party (Simply Apply).
Also reviewing CSV export from Simply Apply to assess feasibility of importing/enriching data in our system.
Key Findings / Observations
Simply Apply strength: fills the top-of-funnel (large volume of job listings, good recommendation engine, constant daily searching).
Simply Apply provides a useful CSV export with columns such as date posted, date applied, applied failure reason, match percentage (relevancy), job title, company, requirements summary and job URL.
The CSV contains ~300 jobs (estimated 250–335), mostly unique companies with occasional duplicates (rarely more than twice).
Our app has richer functionality that Simply Apply lacks: salary field, favorites/preference ranking, role tracker, personality integration, CRM-like job tracking.
Issues observed in current implementation: auto-discovery of positive/negative keywords is broken (defaults to 0 positive and 5 random negative keywords from a sample resume), “auto” discover feature fails to let users add valid positive/negative keywords.
Salary is present in user profile/onboarding but is not surfaced in the Simply Apply CSV UI; may be stored but not presented.
Auto-apply capability exists in Simply Apply but faces complexities (knockout questions and variable application flows) that limit full automation.
Personality assessment currently returns temperament categories (e.g., “rational”) but lacks enough granularity to pick one of four sub-roles without more questions.
Feature Comparison (ours vs. Simply Apply)
Simply Apply:
Excellent at aggregating and surfacing a large set of job listings (top-of-funnel).
Strong recommendation engine (better than LinkedIn in observed cases).
Auto-apply functionality (partial; some jobs have blocking questions).
CSV export with useful metadata (match score, requirements summary, job URL).
Our Role Fairy app:
More comprehensive feature set (salary column, favorites, role tracker, personality integration, CRM-style tracking and richer data model).
Greater downstream value for users (tracking, outreach, contacts enrichment).
Current bugs in job import/auto-discovery features need fixing.
Technical Notes / Opportunities
CSV ingestion approach:
Import Simply Apply CSV into our system (map columns). Use company name -> derive domain -> enrich with likely contacts (e.g., CTO, VP Engineering) -> fetch emails to build outreach columns.
Use job URL to pull full job details where needed.
CSV contains a match percentage (relevancy) that can be used to rank/filter jobs.
Reverse-engineering plan:
Attempt to reproduce Simply Apply functionality quickly (clone their UX/flow in a separate sandbox using Cursor) to validate we can replicate their top-of-funnel capabilities in-house.
If replication is fast and as-good-or-better, prefer in-house to avoid third-party constraints.
Personality assessment refinement:
Current 28-question assessment yields temperament but not precise sub-type. Adding questions or a guided human review (product lead) can refine classification.
Bugs / Issues to Fix
Auto-discovery keyword logic: ensure positive/negative keywords are derived from resume, preferences, or personality fields, not hardcoded or random values.
UI: surface salary in the imported view if available from onboarding/profile.
Auto-apply flow: catalogue which application sites have knockout questions and design partial automation or manual fallback for those cases.
Decisions / Trade-offs Discussed
Prefer building in-house if we can match Simply Apply quickly (fewer dependencies, faster iterations, no external rules/limitations).
Integrating Simply Apply is an option if we cannot reproduce their functionality quickly or if fixing our bugs proves time-consuming; but it introduces external constraints and potential limits on growth velocity.
Proposed Action Items (with owners / timing suggestions)
Short-term: Try to duplicate Simply Apply behavior in a side sandbox (Cursor) and report back with a demo / notes comparing parity and missing features (owner: dev/product lead; timeline: a few days).
Fix auto-discovery keyword bug so positive/negative keywords derive from resume/preferences/personality (owner: engineering; timeline: ASAP).
Add or surface salary column in job imports/view if available (owner: engineering; timeline: ASAP).
Prepare a soft launch demo/marketing push targeted around the 11th (owner: product/marketing; timeline: 2 weeks). Ensure UX is polished for demo audience (tech community).
Use Simply Apply CSV to prototype enrichment pipeline: map company -> domain -> find 2–3 contacts -> fetch emails and append to CSV (owner: data / growth engineering; timeline: prototype in a few days).
Evaluate auto-apply limitations and design fallback strategy for applications with knockout questions (owner: engineering/product; timeline: discovery during CSV reverse-engineer effort).
Consider expanding personality assessment to better discriminate subtypes (owner: product/UX; timeline: next iteration).
Risks / Concerns
Relying on a third-party API (Simply Apply) could constrain product evolution, speed, and control.
Auto-apply automation is inherently brittle due to diverse employer application flows and knockout questions.
If we cannot replicate the top-of-funnel breadth quickly, partnering may be necessary but should be a temporary or constrained integration.
Follow-ups / Next Meeting
Review Cursor sandbox results and decision recommendation (build vs. partner).
Confirm fixes for keyword auto-discovery and salary UI changes.
Prepare soft-launch demo and confirm showable features for tech community outreach.
